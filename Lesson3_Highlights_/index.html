Kamryx Davis

Highlight #1 Version Control System

Version control systems (VCS) allow users to work on files at the same time and go back to previous versions of the files. Files are stored in a repository. Types of version control systems include local version control systems, centralized version control systems, and distributed version control systems. The downside of centralized version control systems and local control version systems is that there is a single point of failure. Distributed version control systems such as Git, Darcs, Bazaar, and Mercurial don’t have this problem. In the distributed control systems each of the users have a complete local copy of the repository data on their computers.

Highlight #2 Git

Git is a distributed version control system that is free and open source. Git was created in 2005. Most of all Git repositories are hosted on GitHub which is the biggest host of git repositories. Git stores information as a snapshot while many others store information by comparing the differences between files. There are 3 main states that files on Git reside in committed, modified, and staged. Some git commands are git init, git add, git commit, git log, git status, git pull, and git push. Git init initializes the repository. Git status checks the status of your files and tells you which state they are in.

Highlight #3 Repository

A repository stores data, they can be remote or local. Remote repositories live on a remote server which is shared with the team members. Local repositories are on the user’s computer. Local repositories can be created by making a brand new repository or by cloning a remote one onto the user’s computer. Git init initializes the repository. Git clone is used to clone existing repositories. If the server that the remote repository is on gets corrupted the clones on client side have the same data and can be used to get the server back to its state before it was corrupted.

Highlight #4 Undo Commits

To undo commits git reset and git revert can be used. Git reset undos commits. It moves the current branch and HEAD back to where the user wants. Git reset can unstage files. Git reset is usually used with one of these 3 flags --soft, --mixed, and --hard. Git revert is used to undo commits that have already been pushed and shared with the team on the repository. Git reset should not be used on commits that have already been pushed and merged with the common repository because it can cause the local repository to become out of sync with the origin repository.

Highlight #5 Remote Server Commands

Some remote server git commands are git fetch, git pull, and git push. In git fetch the remote tracking branches are updated on the local repository. Git fetch can get files from one repository or URL. It can also be used to get files from various repositories at the same time. Git pull pulls the changes from a remote repository to the user’s local repository. When the computer does a git pull the process involves first doing a git fetch and then a git merge or a rebase if specified. Git push pushes the changes from the user’s local repository to the remote repository.
